Every auth: required row from registry/data_source_registry.yaml is listed here together with the environment variable name(s) the system reads at boot. The page tells you what to set and where to get a key. Click verify to live-ping the provider's auth endpoint — the secret value never leaves the server, only the row id is sent across the wire and only an outcome (ok / unauthorized / timeout / …) and the HTTP status come back.

Click set… next to a missing env var to write it to the local .env file (gitignored, mode 0600). Inside a Devin session this button is hidden — Devin secrets are operator-set via the secrets tool and injected into the environment.
